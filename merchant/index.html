    # 商户接口

    # 1. 商户余额查询

    配置不同的发放渠道，会返回不同的账户类型的余额

    # 1.1 商户余额查询接口

    请求地址:

    https://{BASE_URL}/api/Merchant/QueryMerchantAccountBalance

    请求方式: POST

    BASE_URL:

    环境 地址
    测试环境 pre-lsy-outsource-openapi-interface-api.suuyuu.cn
    正式环境 o.linsyun.com

    请求参数 请求参数以JSON方式提交

    参数名 参数说明 长度和属性 必填 备注
    memberCode 商户号 String(50) √ 商户号，参与签名；例如：SH123456
    updateMarkNo 上传标识编号 String(50) √ 上传表示编号，需要全局唯一，参与签名；例如：A001
    timestamp 时间戳 String(20) √ 时间戳，格式为：yyyy-MM-dd HH:mm:ss，与当前时间误差不能超过10分钟，参与签名；例如：2000-01-01 10:10:10
    sign 签名 String(50) √ 签名规则，见文档中说明；例如：D1BD444BFC916C414248CB9BDC5300561208AD03

    响应参数

    参数名 参数说明 长度和属性 必填 备注
    code 状态码 String √ 正常：0，详情请查看【状态码说明】
    message 返回信息描述 String √
    messageType 返回信息类型 Integer √
    data 返回数据 ResponseModel 返回数据

    ResponseModel

    参数名 参数说明 长度和属性 必填 备注
    list 商户账户信息 List<QueryModel> √ 账户信息
    statistics 列表信息 ResponseListInfo √ 总数

    QueryModel

    参数名 参数说明 长度和属性 必填 备注
    accountType 账户类型 String √ 银行账户
    支付宝账户
    充值账户
    accountBalance 账户余额 Decimal

    ResponseListInfo

    参数名 参数说明 长度和属性 必填 备注
    total 总数 Integer √

    参数示例

    # 请求参数示例
{
    "memberCode": "SH123456",
    "updateMarkNo": "A001",
    "timestamp": "2000-01-01 10:10:10",
    "sign": "D1BD444BFC916C414248CB9BDC5300561208AD03",
}

    # 2. 状态码说明

    CODE值 说明
    0 成功
    999 失败
    1000 缺少必要参数
    1001 时间戳格式错误
    1002 时间戳已超过有效期
    1003 上传中请勿重复提交
    1004 商户不存在
    1005 商户配置异常
    1006 签名错误
    1007 上传明细数据超过2000条
    1008 上传标识编号已存在
    1009 未上传数据
    1017 存在未校验通过的数据
    1018 明细数据存在空商户流水号的数据
    1019 全订单明细未上传成功
    1020 未开通支付宝通道，下发类型错误
    1021 未开通银行通道，下发类型错误
    1022 任务不存在
    1023 任务未审核通过
    5000 系统异常
